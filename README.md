Проект **color-follower-ros2-v2** демонстрирует базовые принципы модульной робототехники и компьютерного зрения на платформе ROS 2. В нем реализован автономный мобильный робот, способный находить и следовать за красным кубом в симулированном мире Gazebo.

- **Стек:** Docker, ROS2 Jazzy, Gazebo (Harmonic), OpenCV.
- **Назначение:** Учебный пример для изучения связки симуляции, сенсоров и алгоритмов управления через ROS-топики.

## Состав и структура

| Элемент | Назначение |
|---|---|
| `dockerfile` | Образ контейнера с ROS2 Jazzy Desktop Full и необходимыми пакетами (`ros_gz_bridge`, `rviz2`, OpenCV и др.) |
| `run.sh` | Скрипт быстрого запуска контейнера с пробросом графики |
| `src/mobile_world` | Пакет с SDF-описанием симуляционного мира (`world_v1.sdf`) и launch-файлом для его загрузки |
| `src/mobile_description` | Пакет с URDF-моделью мобильного робота, его сенсорами и launch-файлом |
| `src/red_cube_generator` | Пакет для случайного спавна красного куба в мире |
| `src/red_cube_follower` | Пакет с алгоритмом следования за кубом на основе OpenCV и П-регулятора |
| `docs/` | Гайды по устройству робота (`robot_v1_urdf_guide.md`) и мира (`world_v1_sdf_guide.md`) |

## Архитектура и модульность

Каждый компонент (мир, робот, генератор куба, алгоритм следования) реализован в своем пакете и запускается отдельным launch-файлом. Связь между ними осуществляется исключительно через ROS-топики:

- **Мобильная платформа** получает команды `cmd_vel` от `red_cube_follower`, который анализирует изображение с камеры для поиска красного куба.
- **Генератор куба** спавнит красный куб в случайном месте мира с помощью утилиты `ros_gz_sim create`.
- **Сенсоры** (камера, лидар) публикуют данные через мосты `ros_gz_bridge` в ROS-топики (`/camera/image_raw`, `/lidar`).
- **Визуализация** осуществляется через `rviz2` с готовой конфигурацией.

Такой подход позволяет модифицировать или заменять любой компонент, не затрагивая остальные, что является демонстрацией модульности.

## Алгоритм следования за кубом

Узел `red_cube_follower_node` реализует классический подход компьютерного зрения:

1. **Получение кадра:** Подписка на топик `/camera/image_raw`.
2. **Сегментация цвета:** Преобразование кадра в HSV и создание бинарной маски для красного цвета (два диапазона оттенков).
3. **Морфологическая очистка:** Устранение шумов с помощью эрозии и дилатации.
4. **Поиск контуров:** Выделение самого большого контура, который считается целевым кубом.
5. **Управление движением:** П-регулятор вычисляет угловую скорость на основе смещения центра контура от центра кадра. Если куб не виден или слишком мал, робот вращается на месте для поиска.
6. **Остановка:** Робот останавливается, когда куб занимает более 80% ширины кадра.

## Инструкция по запуску

### Подготовка Docker-образа

```bash
docker build -t ros2_full:v1 .
```

### Запуск в контейнере

Отредактируйте `run.sh`, указав локальный путь к репозиторию (по умолчанию `/home/user/color-follower-ros2-v2`), затем выполните:

```bash
chmod +x run.sh
./run.sh
```

После входа в контейнер соберите проект:

```bash
cd /home/work
colcon build --symlink-install
source install/setup.bash
```

### Запуск симуляции и робота

Запустите все компоненты в отдельных терминалах (или используйте launch-файлы):

```bash
# 1. Запуск мира
ros2 launch mobile_world world.launch.py

# 2. Спавн робота
ros2 launch mobile_description robot.launch.py

# 3. Спавн красного куба
ros2 launch red_cube_generator spawn_red_cube.launch.py

# 4. Запуск следования за кубом
ros2 run red_cube_follower red_cube_follower_node
```

Робот начнет автоматически искать красный куб и следовать за ним.

## Требования

- Ubuntu 24.04 с установленным Docker (или нативная установка ROS2 Jazzy, Gazebo и OpenCV)
- Свободное место на диске для Docker-образа (около 3-4 ГБ)
- Поддержка GPU-ускорения для плавной симуляции (опционально)
